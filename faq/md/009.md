## 快速排查表
先对照这个检查，能解决80%的常见崩溃。

| 症状 | 最可能原因 | 优先尝试 |
| :--- | :--- | :--- |
| 启动即崩 | Java 版本不对 / 内存不足 | 重装Java 17/21，启动器调内存至4-8GB |
| 玩一会崩 | 模组冲突 / 显存爆了 | 二分法查模组，降画质和渲染距离 |
| 进某世界崩 | 区块数据损坏 / 模组缺失 | 备份后删区块，或让管理检查服务器端 |
| 随机崩且有报错 | 驱动打架 / 后台吃资源 | 更新显卡驱动，关浏览器/录屏软件 |

## 标准修复流程

### 🩹 基础急救（每次出问题先做）
1. **增内存**：启动器 → 把分配内存拉到 4~8GB（别超过物理内存一半）。
2. **降画质**：游戏内 → 图形设为“快速”，渲染距离降至12以下。
3. **关后台**：关闭 Chrome、OBS、直播软件等吃内存大户。
4. **验文件**：启动器 → 使用“修复/验证游戏文件”功能。

### 🛠️ 进阶排查（按需深入）

#### 🌋 模组冲突
- **二分定位**：先禁用一半模组，不崩说明问题在另一半，反复缩圈。
- **看日志**：崩溃报告里 `UCH` 开头的那行会出卖出事的模组。
- **版本对齐**：确保所有模组适配当前游戏版本，拒绝跨版本混用。

#### 🖥️ 硬件与驱动
- **显卡面板**：对 javaw.exe 开启高性能模式，关垂直同步。
- **驱动更新**：用显卡官方工具（NVIDIA/AMD App）更新至最新。
- **温度监控**：用工具看CPU/GPU是否过热，过热会触发降频或保护性关断。

#### ☕ Java 和环境
- **换 Java**：高版本游戏必须用 Java 17/21，别用古老的 Java 8。
- **加白名单**：Windows Defender 或杀软把启动器文件夹整个排除。
- **清缓存**：定期清空 `.minecraft/logs` 和 `crash-reports` 文件夹。

## 紧急避险操作
崩溃瞬间保护好你的存档！

1. **立即退出** → 彻底关掉游戏和启动器。
2. **保存日志** → 把 `crash-reports` 里最新的那个文件复制出来。
3. **备份存档** → 立刻复制一份 `.minecraft/saves` 里的世界文件夹。
4. **重启电脑** → 释放占死的资源，重进再试。

## 预防守则
- **只走官道**：模组、整合包从 CurseForge/Modrinth 等官方源下载。
- **常更新**：模组和显卡驱动保持较新版本，别差太多代。
- **勤备份**：重要世界每次下线前手动打包一次，不亏。
- **轻负载**：别边玩边开几十个浏览器标签页，劳逸结合。
